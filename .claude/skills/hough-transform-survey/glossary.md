# Glossary

Alphabetized key terms from the Illingworth–Kittler 1988 survey. Chapter references in brackets.

- **Accumulator (array)** — Multidimensional array of counters, one per parameter cell; incremented by every backprojection that intersects the cell. [[ch01-introduction]]
- **Adaptive Hough Transform (AHT)** — Iterative coarse-to-fine focusing on a small accumulator (typical β = 9 per axis); uses thresholding + connected-components to set the next iteration's parameter limits independently per axis. Illingworth & Kittler [118, 119]. [[ch04-efficient-accumulation]]
- **Autoconvolution (Brown's psf model)** — Under continuous, noise-free conditions the parameter point-spread function of a shape is the autoconvolution of the shape (image of the shape through a pinhole-camera whose aperture is the shape itself). [[ch06-performance-analysis]]
- **Backprojection** — The dual interpretation of the shape-defining relation: one image point produces a hypersurface (one-to-many mapping) in parameter space. Equations 2, 4, 11. [[ch01-introduction]]
- **Backtransform sharpening (Gerig–Klein)** — Re-accumulate after assigning each image point to its single most-supported parameter cell; sharpens peaks. [[ch05-peak-detection]]
- **Bintree** — Rectangular-region space-partitioning data structure proposed by Li et al. [93] to overcome the quadtree's inability to adapt per-axis resolution. [[ch04-efficient-accumulation]]
- **Butterfly peak** — Characteristic shape of the count distribution produced by a true straight line in `(ρ, θ)` space (Leavers & Boyce). Motivates a matched butterfly filter. [[ch05-peak-detection]], [[ch06-performance-analysis]]
- **CHough (Complementary HT)** — Brown's variant where image points contribute positively to some parameter values and negatively to others; cancels sidelobes, sharpens peaks; reduces background mean & variance; more sensitive to quantization. [[ch05-peak-detection]]
- **Connected components (in AHT)** — After thresholding the small accumulator, the shape/extent of connected components dictates how parameter limits are translated, rotated, expanded, or shrunk for the next iteration. [[ch04-efficient-accumulation]]
- **Converging squares algorithm (O'Gorman–Sanderson)** — Pyramid-based peak detector; investigates overlapping hypercube regions from low to high resolution; 8–10× faster than common methods; assumes ≈ cubic peaks. [[ch05-peak-detection]]
- **Dot product space (Sloan)** — Representation for blob-like objects; local shape evidence projected to a transform space for global judgement. [[ch07-ht-and-other-transforms]]
- **Dynamically Quantized Pyramid (DQP)** — Multi-D quadtree with fixed cell count + connectivity; cell boundaries dynamically warped (hierarchical warping) to equalize counts per cell. Order-dependent unless Blanford-rebalanced. [[ch04-efficient-accumulation]]
- **Dynamically Quantized Space (DQS)** — Binary tree of rectangular cells; rules split/merge to keep all cells with ≈ equal evenly-distributed counts; total cell count is an input. [[ch04-efficient-accumulation]]
- **Edge direction (use of)** — The single most cited efficiency lever: constrains the one-to-many backprojection toward one-to-one (foot-of-normal, R-table indexing, circle decomposition). [[ch02-early-development]], [[ch03-shape-parameterizations]]
- **Fast Hough Transform (FHT)** — Multi-D quadtree + hyperplane mapping; incremental plane–cell intersection test (adds and shifts); linear-in-q cost; `O(10³)`/`O(10⁶)` gains for 2-D lines / 3-D planes. Li et al. [75, 76, 94]. [[ch04-efficient-accumulation]]
- **Feature point-spread function (feature psf)** — Pattern of parameter-space counts produced by a single image point. [[ch06-performance-analysis]]
- **Focusing (iterative)** — Coarse-then-fine search; `O(log α / log β)` iterations; saving `O((α/β)^q · log β / log α)`. [[ch04-efficient-accumulation]]
- **Foot of normal (Davies)** — Line parameterization by the intersection of its normal-from-origin with itself; parameter space congruent with image space; one-to-one mapping when edge gradient is reliable. [[ch03-shape-parameterizations]]
- **Generalized Hough Transform (GHT)** — Ballard's extension to arbitrary shapes at arbitrary scale and orientation, keyed on directional edge information; shape stored as a list of boundary points referenced to a reference point. [[ch01-introduction]], [[ch02-early-development]]
- **Goodness G** — `(m_T − m_F) / M_k`. Combined robustness measure of detection. [[ch06-performance-analysis]]
- **Hash/cache accumulator (Brown)** — Replace the accumulator with a small fixed-size content-addressable store; on overflow, garbage-collect by least-votes / age / locality. [[ch04-efficient-accumulation]]
- **Hierarchical warping** — DQP technique that moves cell boundaries to track the mean position of data points → ≈ equal counts per cell. [[ch04-efficient-accumulation]]
- **Hyperplane formulation (Li, Lavin, LeMaster)** — Parameterize shapes so their backprojected surfaces are hyperplanes; plane-vs-hyperrectangle intersection is cheap. [[ch03-shape-parameterizations]], [[ch04-efficient-accumulation]]
- **Hypersurface** — `(q − 1)`-dimensional surface in q-D parameter space defined by an image point's backprojection. [[ch01-introduction]]
- **Image psf** — see Feature point-spread function.
- **Influence function (Thrift–Dunn)** — Heuristic vote weight that decays with the distance between a parameter point and the image feature's parameter hypersurface; lets every image feature increment every accumulator bin. Connects HT to curve fitting. [[ch05-peak-detection]]
- **Muff transform (Wallace)** — Line parameterization by two perimeter exit points `(s₁, s₂)` along the image boundary, `s₁ < s₂`; no transcendentals to compute; matches digital-graphics resolution. [[ch03-shape-parameterizations]]
- **`(m, c)` parameterization** — Slope–intercept; singular as `m → ∞`. [[ch01-introduction]], [[ch03-shape-parameterizations]]
- **`(ρ, θ)` parameterization (Duda–Hart)** — `ρ = x cos θ + y sin θ`; image point ↔ sinusoid; no singularity. [[ch02-early-development]], [[ch03-shape-parameterizations]]
- **P, R, G (Alagar–Thiel)** — Precision `m_T/m_k`, Recall `m_T/M_k`, Goodness `(m_T − m_F)/M_k`. [[ch06-performance-analysis]]
- **Parallel guessing (Fischler–Firschein)** — Compute the HT incrementally and stop once a peak is significant enough; database-architecture analog of focusing. [[ch09-architectures]]
- **Parameter Net (Ballard)** — Connectionist implementation: a feature network + a parameter network with confidence-valued connections. [[ch09-architectures]]
- **Parameter point-spread function (parameter psf)** — Superposition of feature psfs for all of an object's image points; central peak + sidelobes. [[ch06-performance-analysis]]
- **Precision P** — `m_T / m_k`. [[ch06-performance-analysis]]
- **Projection HT** — Computing the HT as a series of projections (Radon-equivalent), often using template-image multiplication + gray-level histograms (Sanz–Dinstein). [[ch07-ht-and-other-transforms]]
- **Radon transform** — Integral transform along all lines; HT ≡ Radon for binary images (Deans 1981). [[ch07-ht-and-other-transforms]]
- **Recall R** — `m_T / M_k`. [[ch06-performance-analysis]]
- **Reference point `(x_r, y_r)`** — Anchor in the GHT template relative to which boundary points and transformations are expressed. [[ch01-introduction]], [[ch03-shape-parameterizations]]
- **R-table (Ballard)** — Boundary-point list indexed by local edge direction; matches image points only to compatibly oriented template entries. [[ch03-shape-parameterizations]]
- **SLAP (Scan-Line Array Processor)** — Linear SIMD array; accumulator cells migrate between PEs to stay co-located with the pixel they accumulate from (Fisher & Highman). [[ch09-architectures]]
- **Subgraph isomorphism interpretation (Kasif et al.)** — GHT is structurally equivalent to subgraph matching via relational sequences. [[ch03-shape-parameterizations]], [[ch08-applications]]
- **Template** — List `{(v_i, w_i)}` of boundary points defining a shape in the GHT framework. [[ch01-introduction]]
- **Variable-slit method (Nagoa–Nakajima)** — Projecting onto edges of a movable rectangular window; related to HT. [[ch07-ht-and-other-transforms]]
- **Voting** — Each image point contributes a count (= 1, weighted, or signed) to all parameter cells consistent with it. [[ch01-introduction]]
